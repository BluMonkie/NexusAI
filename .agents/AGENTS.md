# NEXUS IQ Hackathon Project Rules

## Git Commit Rule
- **Commit to git regularly and meaningfully** after every significant batch of file creation or feature completion.
- Use conventional commit format: `feat:`, `fix:`, `style:`, `refactor:`, `docs:`, `chore:`
- Commit message should describe what was built, e.g.:
  - `feat: add Knowledge Graph page with D3 force-directed visualization`
  - `feat: add AI Copilot chat interface with source citations`
  - `chore: scaffold Vite + React project with design system`
- Commit after each major milestone:
  1. After project scaffolding + design system
  2. After each page/module is completed
  3. After data layer is added
  4. After polish/fixes
- Always run `git add -A && git commit -m "..."` from `c:\Users\Lenovo\Documents\Projects\Hackathon`
